## Layman Summary

Tests if actuarial chain-ladder methods applied to news article relevance scores can predict which corporate risk scenarios materialize, using GDELT news and LLM ground-truth labels.

## Full Summary

The SREDT (Semantic Risk Event Development Triangle) pipeline was fully implemented and executed as follows:

**Method**: 60 European corporate risk scenarios (40 train, 20 test) across energy and financial sectors were generated using hardcoded templates (LLM generation fell back to templates due to free-tier rate limits). For each scenario, GDELT news articles were retrieved with three-tier fallback: gdeltdoc library → direct REST API → locally-generated synthetic article titles. A 60×5 relevance-density mass matrix was constructed where each cell C(i,j) = mean_cosine_sim_above_threshold × log(1 + count_above_threshold), with:
- L0: retrieval volume (log article count)
- L1: sector-level similarity (GICS L1 centroid)
- L2: subsector similarity (GICS L2 centroid)
- L3: risk-category similarity (GARP centroid for scenario's risk type) — observed in train, latent in test
- L4: scenario-specific similarity (scenario text embedding) — observed in train, latent in test

**Venter Actuarial Diagnostics**: Run on 40 training rows per level transition (L0→L1, L1→L2, L2→L3, L3→L4). Coefficient of variation (CV) of individual development ratios determines whether chain-ladder (CV<0.3) or Bornhuetter-Ferguson fallback (CV>0.5) is used for each transition.

**Projection**: Test set L3/L4 values projected using per-transition CL or BF method with sector-specific priors from training set.

**Ground Truth**: 20 test scenarios evaluated by LLM-as-judge (free Llama or Gemini Flash via OpenRouter) — did the scenario materialize given the retrieved articles?

**Baselines**: (1) Flat cosine similarity: mean cosine sim of all articles vs. scenario text (no hierarchy, no threshold); (2) Keyword frequency: fraction of articles mentioning risk-type keywords.

**Metrics**: AUROC, Brier score, Spearman correlation for SREDT vs. both baselines; L0–L4 Spearman rank correlation as circularity check (>0.95 → SREDT collapses to retrieval volume).

**Verdict stored in summary.verdict**: one of 'confirmed' (Spearman improvement >0.15, no circularity), 'partial', 'disconfirmed_proportionality' (all CV>0.5), or 'disconfirmed_circularity' (L0–L4 rank r>0.95).

All 60 scenarios are encoded as exp_gen_sol_out examples with input (scenario text), output (LLM judge verdict + justification for test; SREDT levels for train), and predict_sredt / predict_flat_cosine / predict_keyword fields. Full experimental results including venter_diagnostics, projection_method_used, metrics, and summary are stored in metadata.full_results.
